## 1. نظام الألوان الكامل

اللوحة **ستة رموز فقط**. أي لون خارجها ممنوع. تُعرَّف كلها كـtokens ولا تُكتب قيمة hex مباشرة في أي مكان.

| الرمز | Hex | الدور | متى **لا** يُستخدم |
|---|---|---|---|
| `--white` | `#FFFFFF` | اللون المسيطر: خلفيات الأقسام والبطاقات والحقول | ليس للنص إلا فوق الأخضر أو الحبر |
| `--green` | `#0E3D24` | لون الهوية الأساسي: خلفيات لهجة، الفوتر، شارات، نص عناوين ثانوي | ليس خلفية للصفحة كلها؛ لهجة لا سطح مسيطر |
| `--amber-brand` | `#C8742A` | كهرماني الهوية: الأشكال الكبيرة، الحواف، الرسوم، الخطوط الزخرفية، الأيقونات ≥ 24px | **ممنوع** لأي نص أو زرّ أصغر من 24px على أبيض (تباين 3.5:1 يسقط AA) |
| `--amber-action` | `#A85D1F` | كهرماني الفعل: نص الروابط، نص الأزرار، الأيقونات الدقيقة على الأبيض | ليس للأشكال الزخرفية الكبيرة (أغمق مما يلزم لها) |
| `--ink` | `#141414` | النص الأساسي | ليس خلفية واسعة |
| `--gray` | `#5C5C58` | النص الثانوي، التسميات، النص المساعد، حدود الحقول الهادئة | ليس للنص الأساسي الطويل |

**تباينات محسوبة (مرجع إلزامي):**

| التركيبة | النسبة | الحكم |
|---|---|---|
| `--amber-action` على `--white` | ~5.0:1 | ✅ يمرّ AA للنص العادي |
| `--amber-brand` على `--white` | 3.5:1 | ⚠ عناصر كبيرة/رسوم فقط |
| `--amber-brand` على `--green` | 3.5:1 | ⚠ أشكال كبيرة فقط، لا نص صغير |
| `--green` على `--white` | 12.3:1 | ✅ ممتاز |
| `--ink` على `--white` | ~18:1 | ✅ ممتاز |
| `--gray` على `--white` | 6.7:1 | ✅ يمرّ AA للنص |
| `--white` على `--green` | 12.3:1 | ✅ ممتاز |
| `--white` على `--ink` | ~18:1 | ✅ ممتاز |

**قاعدة الكهرماني الحاسمة:** الكهرماني للنص/الأزرار = `--amber-action` **دائماً**. `--amber-brand` للزخرفة الكبيرة **فقط**. لا استثناء.

**قاعدة الاعتماد على البنية:** يجب أن يقف التصميم بالأبيض/الحبر/الرمادي وحدها. اللون لهجة، لا مُنقِذ لتخطيط ضعيف. (اختبار: غطِّ الأخضر والكهرماني ذهنياً — إن انهار التصميم فالبنية ضعيفة.)

**Tokens (اللون + الظلال + عمود القراءة — كلها معرّفة هنا، لا قيمة مباشرة خارجها):**
```css
:root{
  /* اللون */
  --white:#FFFFFF;
  --green:#0E3D24;
  --amber-brand:#C8742A;
  --amber-action:#A85D1F;
  --ink:#141414;
  --gray:#5C5C58;

  /* الظلال — طبقتان، بلون الحبر 20,20,20 دائماً (لا أسود خالص، لا لون) */
  --shadow-card:   0 1px 2px rgba(20,20,20,0.04),
                   0 4px 12px rgba(20,20,20,0.06);   /* بطاقة/حقل ساكن */
  --shadow-hover:  0 2px 4px rgba(20,20,20,0.05),
                   0 12px 28px rgba(20,20,20,0.10);  /* بطاقة عند hover */
  --shadow-float:  0 4px 8px rgba(20,20,20,0.06),
                   0 16px 40px rgba(20,20,20,0.14);  /* عائم: قائمة/واتساب/modal */
  --shadow-bar:    0 -2px 8px rgba(20,20,20,0.06),
                   0 -8px 24px rgba(20,20,20,0.08);  /* شريط سفلي — معكوس لأعلى */

  /* حدود شعرية بلون الحبر الشفاف (لا رمادي صريح) */
  --hairline:      rgba(20,20,20,0.06);   /* فواصل/حدود خفيفة عامة */
  --hairline-soft: rgba(20,20,20,0.05);   /* فاصل الصورة/النص داخل البطاقة */
  --img-placeholder: rgba(20,20,20,0.04); /* أرضية تحت الصور تمنع وميض التحميل */

  /* عمود القراءة */
  --measure: 640px;   /* ≈ 65 محرفاً — أقصى عرض لأي كتلة نص، حتى داخل حاوية 1200px */
}
```

**قواعد الظلال:** لا `blur` فوق **40px** أبداً (يصبح ضبابياً مولّداً). لا ظل ملوّن (أخضر/كهرماني). الفرق بين `--shadow-card` و`--shadow-hover` محسوس لا مسرحي (الطبقة البعيدة تنتقل من 12px/0.06 إلى 28px/0.10). `--shadow-bar` معكوس لأعلى (`-y`) لأن الشريط يطفو فوق المحتوى القادم من أسفل — تفصيلة يغفلها البناء المولّد. متى يُستخدم كل مستوى موثّق في §7.

## 2. التايبوغرافي

**الخط:** **IBM Plex Sans Arabic** — خط عربي أصيل مرسوم للعربية، مثبّت للمشروع كله بلا بديل. **ممنوع** خط لاتيني مُكيَّف للعربية، و**ممنوع** Cairo وTajawal وInter وPoppins (صاروا default كل موقع عربي ويفضحون الطابع المولّد). للأرقام: أرقام هندية-عربية أو لاتينية حسب المحتوى، لكن **بثبات** في كامل الموقع.

**الأوزان المستخدمة (ثلاثة فقط، لمنع الفوضى):**
- Regular **400** — نص الجسم
- Medium/SemiBold **600** — العناوين الفرعية، الأزرار، التسميات
- Bold **700** — العناوين الكبرى فقط

**سُلّم المقاسات — مدوّر على شبكة 4/8px (الشبكة تعلو على النقاء الرياضي):**

السُلّم: **13 / 16 / 20 / 24 / 32 / 40 / 48 / 56 / 64**. كل قيمة تجلس على شبكة 4px، وكل line-height ينتج ارتفاعاً قابلاً للقسمة على 8 — الانسجام الرأسي الكامل أهم من نسبة 1.25.

| المستوى | الحجم | الوزن | line-height | الارتفاع الناتج | الاستخدام |
|---|---|---|---|---|---|
| `text-xs` | 13px | 400/600 | 1.85 (≈24px) | 24px | تسميات دقيقة، شارات، رسائل خطأ |
| `text-sm` | 16px | 400 | 1.5 (24px) | 24px | نص ثانوي، وصف البطاقة، نص مساعد |
| `text-base` | 20px | 400 | 1.6 (32px) | 32px | نص الجسم الأساسي |
| `text-lg` | 24px | 600 | 1.33 (32px) | 32px | عناوين فرعية، عنوان البطاقة |
| `text-xl` | 32px | 700 | 1.25 (40px) | 40px | عناوين أقسام (موبايل) |
| `text-2xl` | 40px | 700 | 1.2 (48px) | 48px | عناوين أقسام (ديسكتوب) |
| `text-3xl` | 48px | 700 | 1.17 (56px) | 56px | عنوان الـhero (موبايل الأصغر) |
| `text-4xl` | 56px | 700 | 1.14 (64px) | 64px | عنوان hero (موبايل كبير) |
| `text-5xl` | 64px | 700 | 1.13 (72px) | 72px | عنوان الـhero (ديسكتوب) |

> كل المقاسات تُعرّف كـtokens (`--text-xs` … `--text-5xl`) ولا تُكتب قيمة px مباشرة. الأساس 16px = 1rem.

**إعدادات العربية الإلزامية:**
- **line-height:** مضبوط ليجلس على شبكة 8px مع إبقاء تنفّس العربية: نص الجسم 20px/**32px = 1.6** (مريح وقابل للقسمة على 8)، والنص الثانوي 16px/24px = 1.5. لا تستورد 1.4 من إعداد لاتيني — الحدّ الأدنى للنص الطويل 1.6 وللعناوين 1.13–1.33 (كلها تقع على الشبكة).
- **letter-spacing / tracking:** **`0` (normal) للنص العربي دائماً.** tracking السالب أو الموسّع يشوّه اتصال الحروف ويفضح «الطابع المولّد» فوراً. لا `letter-spacing` سالب إطلاقاً على العربية.
- **text-align:** يمين افتراضياً (RTL). العناوين الكبرى قد تُوزَّن يميناً مع كسر تحريري متعمّد.
- **text-wrap:** `pretty` للعناوين لمنع اليتيمة (orphan).
- **font-feature-settings:** فعّل `"liga"` و`"calt"` لضمان الرسم الصحيح للحروف المتصلة.

## 3. نظام المسافات

مقياس مبني على **8px** (مع 4px نصف-درجة). كل هامش/حشو يستخدم هذه الدرجات فقط — لا قيم عشوائية.

| Token | القيمة |
|---|---|
| `space-1` | 4px |
| `space-2` | 8px |
| `space-3` | 12px |
| `space-4` | 16px |
| `space-5` | 24px |
| `space-6` | 32px |
| `space-7` | 48px |
| `space-8` | 64px |
| `space-9` | 96px |
| `space-10` | 128px |
| `space-desktop` | 112px (افتراضي بين الأقسام على الديسكتوب، 14×8) |

**قاعدة اللاتماثل:** الفراغ الأبيض منظَّم حول **محور إزاحة** (asymmetric axis) لا موزَّع بالتساوي. الحشو الداخلي للبطاقات والحقول متماثل؛ أما تخطيط الأقسام فيستخدم إزاحة متعمّدة (مثلاً محتوى بعرض 60% مزاح لليمين مع فراغ 40% لليسار)، لا توسيط آلي دائم.

## 4. الحركة (Motion)

**المنحنى التوقيعي الموحّد — يُطبَّق على كل انتقال بلا استثناء:**
```css
--ease-signature: cubic-bezier(0.2, 0.8, 0.2, 1);
--dur-fast: 180ms;    /* hover، تغيّر حالة صغير */
--dur-base: 220ms;    /* الافتراضي لكل انتقال */
--dur-slow: 300ms;    /* دخول عناصر أكبر (سقف أعلى) */
```

**يُطبَّق على:** hover/focus/active للأزرار والروابط، فتح/إغلاق القائمة، دخول البطاقات عند التمرير، ظهور الشارات، فتح/إغلاق الشريط السفلي.

**قواعد إلزامية:**
- **لا انتقال يتجاوز 300ms.** أبداً.
- **لا `ease` / `ease-in-out` / `linear` افتراضية** — `--ease-signature` فقط.
- الدخول عند التمرير يأتي **من محور اللاتماثل** (إزاحة أفقية خفيفة 16–24px + fade)، لا fade عمودي عام.
- `transform` و`opacity` فقط للحركة (لا تحريك `width/height/top/left`).
- **`prefers-reduced-motion: reduce` محترم** — تُلغى كل الحركات وتبقى الحالة النهائية.

**قائمة الحركات الممنوعة:**
- ❌ fade-in/slide-up عام افتراضي على كل شيء (توقيع الـAI الأول).
- ❌ حركات متتالية (stagger) طويلة > 60ms بين العناصر.
- ❌ bounce / elastic / overshoot مبالغ.
- ❌ parallax ثقيل أو حركة مربوطة بالتمرير (scroll-jacking).
- ❌ دوران/تكبير مستمر (infinite spin/pulse) كزينة.
- ❌ typewriter effect على النص العربي.
- ❌ مدد طويلة (> 300ms) أو `ease` بطيء يوحي بالبطء «السينمائي» المولّد.

## 5. الأبعاد والمقاسات الرقمية

### 5.1 نقاط الكسر (Breakpoints)
| الاسم | المدى |
|---|---|
| Mobile | `0 – 767px` |
| Tablet | `768px – 1023px` |
| Desktop | `≥ 1024px` |
| (Wide اختياري) | `≥ 1440px` — الحاوية تتوقّف عن التمدّد |

### 5.2 الحاوية
- **العرض الأقصى للحاوية:** `1200px`.
- الحشو الجانبي (gutter): **موبايل 20px** · **تابلت 32px** · **ديسكتوب 40px**.
- المحتوى يُتوسَّط أفقياً داخل 1200px، مع إزاحة لا-متماثلة **داخل** الشبكة لا على الحاوية نفسها.

### 5.3 الهيدر
- ارتفاع: **موبايل 64px** · **ديسكتوب 80px**.
- ثابت أعلى الصفحة (sticky)، خلفية `--white`، حدّ سفلي 1px `--gray` بشفافية منخفضة عند التمرير فقط.
- الشعار يمين، التنقّل يسار (RTL).

### 5.4 الـHero
- الارتفاع: **موبايل `85vh`** (بحدّ أدنى 560px) · **ديسكتوب `90vh`** (بحدّ أقصى 900px).
- **لا `100vh`** (يتجاوز شريط المتصفح على الموبايل ويقصّ المحتوى).

### 5.5 البطاقات (بطاقة الصنف/المنتج)
الهدف: بطاقة تُحسّ كجسم ملموس مصمّم بيد، لا مستطيل جامد.

| العنصر | القيمة | التعليل |
|---|---|---|
| عرض البطاقة | **~368px** (شبكة 3× داخل 1200px، فجوة 32px) | تابلت: عمودان فجوة 24px · موبايل: عمود واحد |
| **نسبة الصورة** | **3:2 → 368×245px** | أرشق من 4:3؛ يعطي إيقاعاً رأسياً أفضل |
| موضع الصورة | **full-bleed لأعلى البطاقة**، تلامس الحواف العلوية/الجانبية، بلا حشو حولها | الحشو لمنطقة النص فقط |
| زوايا الصورة | العلويتان 12px (تطابق البطاقة)، السفليتان 0 | تلتقي بمنطقة النص |
| **حشو منطقة النص (غير موحّد)** | **20px جانبي · 20px سفلي · 18px علوي** | العلوي أقل لأن الصورة فوقه تعطي وزناً؛ اللاتماثل الطفيف = يد خبير |
| شارة التوفّر | **داخل منطقة النص، أول عنصر فوق العنوان**، هامش سفلي 12px | ليست فوق الصورة (تشوّشها)، بل كسطر تصنيف |
| العنوان | `text-lg` (24px) وزن 600، هامش سفلي **8px**، line-height 1.33 | |
| الوصف | `text-sm` (16px) وزن 400 `--gray`، line-height 1.5، **سطران حدّ أقصى** (`-webkit-line-clamp:2`) | القصّ يوحّد ارتفاع البطاقات = شبكة منضبطة |
| فاصل قبل الفعل | هامش علوي **16px** | |
| صفّ الفعل السفلي | رابط/سعر يمين + أيقونة/سهم يسار، `align-items:center` | الأرقام `tabular-nums` |
| الحاشية بين الصورة والنص | `border-top: 1px solid var(--hairline-soft)` (rgba(20,20,20,0.05)) أسفل الصورة مباشرة | إحساس «طبقتين مبنيّتين» لا لوح واحد |
| **ارتفاع البطاقة** | موحّد بالقصّ لا بـ`height` ثابت: صورة 245px + نص مقصوص ≈ 400px | نسبة ~1.09:1 — مربّع مائل رأسياً قليلاً، الأرقى للبطاقات |
| نصف قطر البطاقة | **`--radius-card` = 12px** | |

**hover (ديسكتوب فقط):** رفع البطاقة `translateY(-3px)` + `--shadow-hover`، و**تكبير الصورة داخلها `scale(1.03)`** مع `overflow:hidden` على حاوية الصورة — الصورة تتنفّس داخل إطارها الثابت، بالمنحنى التوقيعي `--dur-base`. `will-change: transform` على البطاقة القابلة للـhover فقط.

### 5.6 الأزرار
- الارتفاع: **48px** (أساسي) · **40px** (ثانوي صغير).
- **هدف اللمس الأدنى: 44×44px** — أي زر أصغر بصرياً يأخذ hit-area شفافة 44px.
- الحشو الأفقي: **24px**.
- نصف القطر: **`--radius-btn` = 8px**.
- حجم نص الزر: 16px وزن 600.

### 5.7 حقول الفورم
- الارتفاع: **52px**.
- الحشو الداخلي: **16px أفقي**.
- نصف القطر: **`--radius-field` = 8px**.
- الحدّ: 1px `--gray`؛ عند focus: 2px `--green`.
- حجم النص: 16px (يمنع تكبير iOS التلقائي).
- عرض: كامل عرض الحاوية/العمود.
- تسمية الحقل فوقه، 13px وزن 600 `--gray`.

### 5.8 الصور
- Hero: full-bleed أو ضمن الشبكة اللا-متماثلة بنسبة **16:9** (ديسكتوب) / **4:5** (موبايل).
- بطاقات: **3:2** (368×245px). ثمبنيلات/معرض: **1:1**.
- كل الصور `object-fit: cover` وأبعادها معلومة مسبقاً (width/height) لمنع CLS.
- **أرضية تحت كل صورة** `background: var(--img-placeholder)` (rgba(20,20,20,0.04)) فتظهر الصورة بـfade على أرضية بدل وميض أبيض عند التحميل الكسول.
- **الصور تبقى بألوانها الطبيعية الكاملة — لا `saturate` ولا تعديل تشبّع.** خضار الطزاجة الزاهي هو مصدر الحيوية والصدق.
- يُسمح فقط بطبقة `::after` خفيفة جداً (تدرّج من شفاف لأسفل حتّى `rgba(20,20,20,0.04)`) **فقط إن وُضع نص/شارة فوق الصورة** — لتثبيته لا لتعديل الصورة نفسها.

### 5.9 الأيقونات
- الأساسي: **24×24px**. الصغير: **20px**. الكبير (زخرفي): **32–48px**.
- stroke موحّد (مثلاً 1.75px)، مجموعة أيقونات واحدة فقط.

### 5.10 المسافات بين الأقسام (رأسياً) — سُلّم لا رقم موحّد
التوحيد نفسه يقع في فخّ التناظر الآلي — التنويع مقصود.

| الشاشة | بين الأقسام |
|---|---|
| Mobile | **64px** (`space-8`) |
| Tablet | **96px** (`space-9`) |
| Desktop — افتراضي | **112px** |
| Desktop — بين الأقسام الكبرى فقط (Hero←الأصناف، الأصناف←لماذا نحن) | **128px** (`space-10`) |
| أقسام فرعية داخل الكتلة الواحدة | **80px** |

> 112px يحتاج token جديد (`--space-desktop` = 112px) خارج سُلّم 8px الأساسي؛ 112 = 14×8 فيبقى على الشبكة. حشو داخل القسم (أعلى/أسفل) يطابق الجدول أو 3/4 منه للأقسام الفرعية.

### 5.11 الزر العائم (واتساب) والشريط السفلي
- **الزر العائم (واتساب):** 56×56px دائري، ثابت أسفل **يسار** الشاشة (RTL)، على بعد 20px من الحافتين، `z-index` أعلى من المحتوى وأدنى من القائمة الكاملة. **موبايل فقط** (انظر §9).
- **الشريط السفلي الثابت (CTA العيّنة):** ارتفاع **64px** + safe-area، خلفية `--white`، حدّ علوي 1px، يحوي زر «اطلب عيّنة» بعرض شبه كامل. ثابت أسفل الشاشة على الموبايل، يختفي عند فتح القائمة الكاملة.

## 6. بنية الصفحة قسماً قسماً (بالترتيب)

الموقع **صفحة واحدة** بكل الأصناف مكشوفة (انظر §9). الترتيب:

1. **الهيدر** — شعار (يمين) + تنقّل (يسار) على الديسكتوب / أيقونة هامبرغر على الموبايل.
2. **الـHero** — عنوان (`text-5xl` ديسكتوب / `text-4xl` موبايل) + سطر داعم (`text-lg` `--gray`) + زر أساسي واحد. تخطيط لا-متماثل: نص مزاح لجهة، صورة/عنصر بصري للجهة الأخرى.
3. **شريط الثقة/القيمة** — 3–4 نقاط قيمة قصيرة بأيقونات 24px.
4. **الأصناف / المنتجات** — شبكة بطاقات (كل الأصناف مكشوفة، لا تبويب مخفي)، عنوان قسم + وصف مزاح لا-متماثلاً.
5. **قسم «لماذا ألياف الشمال»** — كتلة لا-متماثلة: صورة كبيرة جهة + نص/نقاط جهة، خلفية لهجة `--green` أو أبيض.
6. **عيّنة / اطلب عيّنة** — CTA بارز، مرتبط بالشريط السفلي على الموبايل.
7. **تواصل / فورم** — حقول (§5.7) + معلومات تواصل + واتساب.
8. **الفوتر** — خلفية `--green`، نص `--white`، روابط + حقوق. ارتفاع مريح، `space-8/9`.

كل قسم يفصله عن التالي المسافة الرأسية في §5.10 حسب الشاشة.

## 7. مكوّنات الواجهة بكل حالاتها

### 7.1 الزر
| الحالة | المظهر |
|---|---|
| **Primary — default** | خلفية `--green`، نص `--white`، ارتفاع 48px، radius 8px |
| Primary — hover | خلفية أغمق قليلاً من الأخضر (`--green` -6% lightness)، انتقال `--dur-fast` |
| Primary — active | تعتيم طفيف + `transform: translateY(1px)` |
| Primary — focus | حلقة focus 2px `--amber-action` بإزاحة 2px — عبر `:focus-visible` **فقط** (للوحة المفاتيح لا الفأرة) |
| Primary — disabled | خلفية `--gray` بشفافية 40%، لا مؤشر |
| **Secondary — default** | خلفية شفافة، حدّ 1.5px `--green`، نص `--green` |
| Secondary — hover | خلفية `--green` بشفافية 6% |
| **Text/Link button** | نص `--amber-action`، خطّ سفلي عند hover |

### 7.2 حقل الفورم
| الحالة | المظهر |
|---|---|
| default | حدّ 1px `--gray`، خلفية `--white`، نص `--ink` |
| focus | حدّ 2px `--green`، بلا ظل توهّج |
| filled | نص `--ink` بوضوح كامل |
| error | حدّ 2px بلون خطأ مشتق من الكهرماني الداكن `--amber-action`، رسالة 13px تحته |
| disabled | خلفية رمادية فاتحة، نص `--gray` |
| placeholder | `--gray` |

### 7.3 البطاقة
- default: خلفية `--white`، radius 12px، `box-shadow: var(--shadow-card)`، حدّ 1px `var(--hairline)`. الصورة full-bleed لأعلى البطاقة مع hairline فاصل (§5.5).
- hover (ديسكتوب فقط): `translateY(-3px)` + `var(--shadow-hover)` + تكبير الصورة `scale(1.03)`، بـ`--dur-base`. الفرق بين card وhover محسوس لا مسرحي.
- لا ظلال ملوّنة، لا توهّج، لا blur > 40px.

### 7.4 شارة التوفّر (Availability badge)
- **متوفّر:** خلفية `--green` شفافية 10%، نص `--green`، نقطة 8px `--green`.
- **قريباً / محدود:** خلفية `--amber-brand` شفافية 12%، نص `--amber-action`، نقطة `--amber-brand`.
- **غير متوفّر:** خلفية `--gray` شفافية 12%، نص `--gray`.
- ارتفاع 24px، حشو أفقي 10px، radius **pill (999px)** — استثناء متعمّد لتنويع الزوايا (انظر أدناه).
- نص `text-xs` وزن 600.

**قاعدة تنويع الزوايا (مضادة للطابع المولّد):** لا تُوحَّد كل الزوايا على رقم واحد. النظام: أزرار/حقول **8px**، بطاقات **12px**، شارات **pill**، صور **12px**. هذا تدرّج مقصود لا رقم واحد مكرّر.

### 7.5 لمسات المصمّم الخبير (micro-details — تُطبّق أينما وُجدت الحالة)
- **فواصل/حدود شعرية بلون الحبر الشفاف لا الرمادي الصريح:** كل فاصل/حدّ خفيف = `var(--hairline)` (rgba(20,20,20,0.06)) لا `--gray` صريح. الحدّ الرمادي الصريح يبدو «مرسوماً»؛ الحبر الشفاف يبدو «ظلاً للحافة» — أنعم وأرقى.
- **خطّ تحريري كهرماني قصير فوق عناوين الأقسام:** خطّ أفقي 40px بسمك 3px بلون `--amber-brand`، أو كلمة تصنيف صغيرة (`text-xs` وزن 600 `--amber-action`) قبل العنوان. يوظّف الكهرماني كنبضة واحدة لكل قسم.
- **`:focus-visible` بدل `:focus`:** حلقة `--amber-action` بإزاحة 2px تظهر للوحة المفاتيح لا للفأرة — تفصيلة وصول يغفلها البناء المولّد.
- **أرضية تحت الصور (`--img-placeholder`):** تمنع وميض التحميل (§5.8).
- **`tabular-nums` للأرقام في البطاقات:** `font-variant-numeric: tabular-nums` للأسعار/الأرقام كي تصطفّ عمودياً.
- **تدرّج سماكة الحدّ:** الحقل عند focus 2px، الحدّ الهادئ 1px، حدّ الصورة hairline — تنويع سماكة مقصود لا رقم واحد.
- **حالات انتقالية (Loading/Empty):** بطاقات skeleton بـ`--img-placeholder` مع shimmer خفيف جداً بالمنحنى التوقيعي؛ حالة «لا نتائج» بنص `--gray` وأيقونة 32px هادئة. غيابها يكشف البناء السطحي.

## 8. قواعد RTL و bidi

- `dir="rtl"` و`lang="ar"` على `<html>`.
- التخطيط كله مبني بـ **logical properties**: `margin-inline-start/end`, `padding-inline`, `inset-inline`, `border-inline` — لا `left/right` صريحة.
- **الأيقونات الاتجاهية تُعكس** (أسهم «التالي/السابق»، chevron، أيقونة الرجوع). الأيقونات غير الاتجاهية (بحث، قلب، سلة) **لا تُعكس**.
- الأرقام والمقاطع اللاتينية/الإنجليزية داخل نص عربي تُعالَج بـ `unicode-bidi: plaintext` أو `<bdi>` لمنع انقلاب ترتيب أرقام الهاتف/الأسعار.
- الشعار يمين، التنقّل يسار. الزر العائم يسار. حلقة focus وإزاحتها تحترم الاتجاه.
- المحاذاة الافتراضية للنص: يمين. الأعمدة تتدفّق من اليمين لليسار.
- علامات الترقيم واتجاه الاقتباسات عربية صحيحة.

## 9. سلوك الموبايل الكامل (قرارات محسومة)

- **قائمة هامبرغر بملء الشاشة (full-screen overlay):** عند النقر تفتح طبقة تغطّي الشاشة كاملة، خلفية `--white`، روابط كبيرة (`text-lg`، هدف لمس ≥ 48px)، زر إغلاق واضح. لا drawer جانبي جزئي.
- **واتساب عائم فقط:** الزر العائم (56px، أسفل يسار) هو قناة التواصل السريع على الموبايل — لا يتكرّر كعنصر ثابت آخر.
- **شريط سفلي ثابت للعيّنة:** CTA «اطلب عيّنة» ثابت أسفل الشاشة (64px + safe-area)، يظهر بعد تمرير الـhero، يختفي عند فتح القائمة الكاملة.
- **صفحة واحدة بكل الأصناف مكشوفة:** لا تبويب يخفي أصنافاً، لا accordion يطوي المحتوى الأساسي. كل الأصناف مرئية بالتمرير.
- **إعادة تصميم اللاتماثل للعمود الضيّق:** لا يُطوى التخطيط اللا-متماثل آلياً إلى stack متماثل. الزجزاج الأفقي (§11.2) يتحوّل على الموبايل إلى **إزاحة عمودية طفيفة + حافّة نازفة واحدة ليمين الشاشة** (العنصر البصري يخرج ~16px عن الحافة بدل التوسيط) — يحفظ التوتر في عمود ضيّق.
- **حماية تقنية من الـscroll الأفقي:** `overflow-x: clip` على الحاوية الأمّ (لا `hidden` — يكسر sticky)، ويُبنى النزيف بـ`margin-inline-end` سالب محسوب لا بـ`transform`.
- أهداف اللمس ≥ 44px، والتباعد بين الأهداف ≥ 8px.
- `font-size: 16px` كحدّ أدنى لحقول الإدخال لمنع تكبير iOS.
- احترام `safe-area-inset` للعناصر الثابتة (الشريط السفلي والزر العائم).

## 10. قواعد الأداء

- **الصور:** WebP/AVIF مع fallback، `width/height` صريحان دائماً (CLS ≈ 0)، `loading="lazy"` لما تحت الطيّة، الـhero `fetchpriority="high"`.
- **الخطوط:** `font-display: swap` أو preload للخط الأساسي، مجموعة أوزان محدودة (400/600/700 فقط)، subset عربي.
- **CLS:** الهدف **< 0.05**. كل صورة/إطار محجوز بأبعاده.
- **LCP:** الهدف **< 2.5s**. عنصر الـhero (نص أو صورة) هو LCP ويُحمَّل بأولوية.
- **الحركة:** `transform`/`opacity` فقط، 60fps، لا حركة تُجبر إعادة layout.
- **JS:** أدنى قدر، لا مكتبات ثقيلة لتأثيرات بسيطة. لا framework إن لم يلزم.
- **لا** خطوط أيقونات ثقيلة كاملة — SVG inline أو subset فقط.
- CSS حرج inline للطيّة الأولى إن أمكن.

## 11. عنصر التميّز

**محور اللاتماثل التحريري + محور الفراغ الأبيض المنظَّم** هو توقيع الهوية البصري الذي يرفعها فوق القوالب. يتجسّد في:
- إزاحة المحتوى عن مركز الشبكة بنسبة مقصودة (لا توسيط آلي).
- حافّة نازفة (full-bleed) لعنصر بصري واحد لكل قسم رئيسي يكسر حدّ الحاوية.
- تباين حجم متعمّد بين العنوان الكبير والنص الصغير (قفزة من `text-5xl` إلى `text-base`).
- الكهرماني كلمسة نبض واحدة لكل مساحة رؤية — لا تكرار كثيف يبدّده.

### 11.1 عمود القراءة المحدود
كل كتلة نص لها **`max-width: var(--measure)` = 640px (≈ 65 محرفاً)**، حتّى داخل حاوية 1200px. النص لا يمتدّ كامل العرض أبداً؛ يجلس في عمود قراءة مريح مزاح لا-متماثلاً. الفراغ الأبيض الناتج على الجهة مقصود لا منسيّ.

### 11.2 نظام التثبيت اللا-متماثل (إيقاع يقود التمرير)
الفراغ الجانبي المقصود يُملأ بـ**عنصر تثبيت بصري واحد لكل قسم** (رقم قسم كبير باهت `--img-placeholder`، أو خطّ عمودي كهرماني رفيع، أو صورة نازفة للحافة).
- **قاعدة الإيقاع:** عنصر تثبيت واحد فقط لكل قسم، **تتناوب جهته** (قسم يمين، التالي يسار) — زجزاج يقود عين القارئ لأسفل. الأبيض المسيطر + التناوب = سرد بصري.

### 11.3 المبدأ الحاكم الواحد
«**الفراغ الأبيض هو البطل، وكل عنصر آخر ضيف يبرّر حضوره**». قبل أي عنصر جديد يُسأل: هل يبرّر أخذه من مساحة الأبيض؟ فإن لا — يُحذف. الظل خفيف لأن البطل أبيض؛ اللاتماثل لأن التوسيط يقتل البطل؛ الكهرماني نبضة واحدة لئلا يزاحمه.

هذا العنصر يجب أن يُبنى **أولاً وفارغاً** (§SOP المرحلة 1): إن لم يُحسّ التوتر بلا محتوى، فاللاتماثل مزيّف.

## 12. المحرّمات البصرية والكودية (كاملة)

**بصرياً:**
- ❌ تدرّجات لونية مشبعة، خاصة الأزرق/البنفسجي الكهربائي.
- ❌ أي لون خارج الرموز الستة.
- ❌ استخدام `--amber-brand` لنص أو زر صغير.
- ❌ توسيط كل شيء آلياً (يقتل اللاتماثل).
- ❌ زوايا موحّدة على رقم واحد لكل العناصر.
- ❌ ظلال ثقيلة/ملوّنة أو blur > 40px أو توهّج. (نظام الطبقتين في §1 مسموح ومطلوب — الممنوع هو الظل الافتراضي الكسول والملوّن.)
- ❌ خط لاتيني مُكيَّف للعربية، أو أكثر من عائلة خط واحدة.
- ❌ `letter-spacing` سالب على العربية.
- ❌ إيموجي كعناصر واجهة.
- ❌ رسوم/أيقونات بأنماط مختلطة (stroke + fill عشوائي).

**كودياً:**
- ❌ قيم hex/px مكتوبة مباشرة — كله عبر tokens.
- ❌ `left/right` بدل logical properties.
- ❌ `100vh` للـhero.
- ❌ `ease`/`ease-in-out` افتراضية بدل `--ease-signature`.
- ❌ تحريك `width/height/top/left`.
- ❌ صور بلا `width/height`.
- ❌ font-size < 16px لحقول الإدخال.
- ❌ حركات بلا احترام `prefers-reduced-motion`.
- ❌ مكتبات JS ثقيلة لتأثيرات بسيطة.

## 13. أخطاء الطابع المولّد التي يجب اصطيادها — Checklist نهائي

> يُدقَّق على المخرج قبل الاعتماد. كل بند: الخطر · لماذا يفضح نفسه لعين تقنية · القاعدة المضادة الدقيقة.

**□ 1. الحركة الافتراضية العامة**
- الخطر: fade-in/slide-up موحّد على كل عنصر بـ`ease` ومدة ~500–800ms.
- يفضح نفسه: هو التوقيع البصري الأول لمولّدات المواقع؛ العين التقنية تتعرّف عليه فوراً.
- القاعدة المضادة: `--ease-signature` فقط، مدة ≤ 300ms، الدخول من محور اللاتماثل، وليس على كل عنصر — بل على كتل مختارة.

**□ 2. المسافات المتناظرة آلياً / التوسيط الشامل**
- الخطر: كل قسم موسَّط، فراغ علوي = سفلي، أعمدة متساوية دائماً.
- يفضح نفسه: الإيقاع «الميكانيكي» المثالي لا يشبه يد مصمّم.
- القاعدة المضادة: محور إزاحة لا-متماثل مقصود لكل قسم رئيسي، وحافّة نازفة واحدة، وتباين حجم متعمّد.

**□ 3. الزوايا الموحّدة على رقم واحد**
- الخطر: `border-radius: 8px` (أو 16px) على كل شيء: أزرار، بطاقات، صور، شارات، حقول.
- يفضح نفسه: التوحيد المطلق علامة قالب مولّد.
- القاعدة المضادة: تدرّج مقصود — أزرار/حقول 8px، بطاقات/صور 12px، شارات pill.

**□ 4. الظلال المولّدة**
- الخطر: ظل ناعم كبير موحّد (`0 10px 30px rgba(0,0,0,0.1)`) على كل بطاقة، أو ظلال ملوّنة/توهّج.
- يفضح نفسه: «ظل Tailwind الافتراضي» معروف بصرياً.
- القاعدة المضادة: نظام ظل من طبقتين بلون الحبر (`--shadow-card`/`hover`/`float`/`bar`)، blur ≤ 40px، لا لون، والفرق بين المستويات محسوس لا مسرحي.

**□ 5. الخط اللاتيني المكيّف للعربية**
- الخطر: استخدام Inter/Roboto/Poppins مع fallback عربي، أو خط لاتيني «يدعم» العربية برسم رديء.
- يفضح نفسه: أشكال الحروف والاتصالات والتنقيط تبدو خاطئة لعين عربية، وهذا أوضح فضيحة للطابع المولّد في موقع عربي.
- القاعدة المضادة: خط عربي أصيل واحد (IBM Plex Sans Arabic فقط — لا Cairo/Tajawal)، line-height مضبوط على شبكة 8px (نص الجسم 1.6، الثانوي 1.5)، `letter-spacing: 0`.

**□ 6. tracking / line-height مستورد من إعداد لاتيني**
- الخطر: `letter-spacing: -0.02em` وline-height 1.4 منقولان من قالب لاتيني.
- يفضح نفسه: النص العربي يبدو مضغوطاً ومتلاصقاً وغير مقروء.
- القاعدة المضادة: `letter-spacing: 0`، line-height مضبوط على شبكة 8px (نص الجسم 1.6، الثانوي 1.5، العناوين 1.13–1.33 كما في §2) — لا قيم لاتينية مضغوطة (1.4).

**□ 7. أسماء المتغيرات/الكلاسات العامة المولّدة**
- الخطر: `.container`, `.wrapper`, `.card`, `.btn-primary`, `--color-1`, أسماء عشوائية أو مولّدة.
- يفضح نفسه: بنية الكود نفسها تكشف الأصل المولّد عند الفحص.
- القاعدة المضادة: تسمية دلالية متّسقة مرتبطة بالهوية (`--green`, `--amber-action`, `--space-6`, `.availability-badge`)، ونظام تسمية واحد (BEM أو ما يماثله) بثبات.

**□ 8. اللون المفرط / التدرّجات**
- الخطر: تدرّجات، ألوان لهجة متعدّدة، خلفيات ملوّنة كثيرة.
- يفضح نفسه: انفلات اللوحة علامة قالب مولّد يحاول «الإبهار».
- القاعدة المضادة: ستة رموز فقط، أبيض مسيطر، الأخضر لهجة والكهرماني نبضة واحدة لكل مساحة رؤية.

**□ 9. المحتوى النائب المثالي**
- الخطر: صور stock عامة، نص lorem، أرقام إحصائية مخترعة، أيقونات زائدة بلا معنى.
- يفضح نفسه: «الكمال الفارغ» يوحي بقالب مملوء آلياً.
- القاعدة المضادة: محتوى حقيقي، لا حشو، لا أرقام/أيقونات بلا وظيفة.

**□ 10. الشبكة المثالية الجامدة**
- الخطر: 3 بطاقات متطابقة تماماً في كل صفّ بلا أي تنويع بصري.
- يفضح نفسه: الرتابة التامة سمة مولّدة.
- القاعدة المضادة: نظام موحّد للبطاقات لكن مع تنويع محتوى/صورة حقيقي، وقسم واحد على الأقل يكسر الشبكة (لا-متماثل).

**□ 11. الأيقونات الاتجاهية غير المعكوسة في RTL**
- الخطر: أسهم «التالي» تشير يميناً في واجهة RTL.
- يفضح نفسه: خطأ RTL كلاسيكي لقالب لاتيني مترجَم آلياً.
- القاعدة المضادة: عكس الأيقونات الاتجاهية فقط، واختبار كل سهم/chevron.

**□ 12. `100vh` وكسر الموبايل**
- الخطر: hero بـ`100vh` يُقصّ بشريط متصفح الموبايل، أو CLS من صور بلا أبعاد.
- يفضح نفسه: القفزات والقصّ تجربة «غير مصقولة» تُقرَن بالبناء الآلي المتعجّل.
- القاعدة المضادة: `85–90vh` بحدود، أبعاد صور صريحة، `svh` عند اللزوم.

**□ 13. حالات المكوّن الناقصة**
- الخطر: أزرار/حقول بلا hover/focus/active/disabled/error حقيقية.
- يفضح نفسه: غياب الحالات يكشف بناءً سطحياً مولّداً.
- القاعدة المضادة: تنفيذ كل الحالات في §7 حرفياً، وحلقة focus مرئية دائماً.

**□ 14. الرتابة الحركية بلا `reduced-motion`**
- الخطر: حركات تعمل دائماً بلا احترام تفضيل المستخدم.
- القاعدة المضادة: `@media (prefers-reduced-motion: reduce)` يلغي كل الحركات.

## 14. اعتراضات وتحسينات مقترحة

الحزمة **سليمة ومتماسكة في جوهرها**، ولا أختلق اعتراضات. أقوى قراراتها — الأبيض المسيطر، محور اللاتماثل، انضباط اللونين — صحيحة وأبقيها دون مساس. الملاحظات التالية فقط:

**① تفريع الكهرماني — تصحيح إلزامي.**
- القرار الأصلي: استخدام `#C8742A` كلون نبض واحد للكل.
- لماذا خطأ: تباينه على الأبيض 3.5:1، يسقط AA لأي نص/زر عادي — عيب وصول حقيقي لا تفضيل.
- البديل الدقيق: `--amber-action #A85D1F` للنص/الأزرار (~5:1)، و`--amber-brand #C8742A` للزخرفة الكبيرة فقط. (مُدمج في §1.)
- القوة: **تصحيح إلزامي.**

**② line-height و letter-spacing العربيان — تصحيح إلزامي إن كانا مستوردين.**
- القرار المحتمل: قيم منقولة من إعداد لاتيني.
- لماذا خطأ: العربية تحتاج تنفّساً رأسياً أكبر وtracking صفرياً؛ الاستيراد يشوّه القراءة ويفضح الطابع المولّد.
- البديل: line-height ≥ 1.7، `letter-spacing: 0`. (مُدمج في §2.)
- القوة: **تصحيح إلزامي.**

**③ قرار موبايل مستقل للّاتماثل — تحسين قوي.**
- القرار الأصلي: الحفاظ على اللاتماثل على الموبايل (سليم كمبدأ).
- الملاحظة: يحتاج تعريفاً رقمياً صريحاً (كيف يُترجَم على 375px) وإلا انزلق لطيّ آلي عند التنفيذ.
- البديل: قواعد §9 (إزاحة/بليد/تدرّج حجم) بدل stack متماثل.
- القوة: **تحسين قوي.**

**④ منحنى حركة توقيعي موحّد وموثّق — تحسين قوي.**
- الملاحظة: الحركة أخطر نقطة على «لا طابع AI»؛ يجب تثبيت منحنى ومدّة واحدين رقمياً وتطبيقهما بلا استثناء.
- البديل: §4.
- القوة: **تحسين قوي.**

**⑤ سقف الكهرماني الإشراقي — تفضيل فقط (لا يُعاد فتحه).**
- ملاحظة: يمكن دفع الكهرماني نحو زعفران أكثر إشراقاً لطاقة أعلى.
- لماذا لا أوصي: يخسر الرزانة التراثية التي هي من أقوى نقاط الهوية.
- القوة: **تفضيل فقط — أبقِ الكهرماني المكتوم.**

**الخلاصة:** تصحيحان إلزاميان (① و②)، تحسينان قويان (③ و④)، وتفضيل واحد لا يستحق إعادة النظر (⑤). ما عدا ذلك الحزمة جاهزة للبناء كما هي.

## 15. تعارضات جرى حلّها

- **الكهرماني الواحد مقابل التفريع:** المرفقات الأولى تعامل `#C8742A` كلون نبض واحد؛ المراجعة النهائية فرّعته. **الحلّ المعتمد هنا:** التفريع (§1) — لأنه يحلّ عيب تباين مُقاس.
- **قيم line-height العربية:** حيث لم تُحدَّد صراحةً في المرفقات، ضُبطت لتجلس على شبكة 8px مع إبقاء تنفّس العربية (نص الجسم 1.6، العناوين 1.13–1.33) و`letter-spacing: 0`.
- **ارتفاع الـhero:** حيث ذُكر `vh` دون رقم، ثُبّت على 85vh (موبايل) / 90vh (ديسكتوب) بحدود دنيا/قصوى، وحُظِر `100vh` لسبب تقني (§5.4).
- **أي قيمة رقمية لم تُذكر صراحةً في المرفقات** (بعض المقاسات الدقيقة) اشتُقّت من نظام المسافات 8px والسُّلّم المدوّر على الشبكة حفاظاً على الاتساق، ولم تُخترَع قيم خارج هذين النظامين.

### قرارات المراجعة النهائية (تعميق التفاصيل والجمالية)
- **سُلّم التايبوغرافي مدوّر على شبكة 4/8px:** استُبدل سُلّم نسبة 1.25 (12.8/25/31/39/49/61 لا تجلس على الشبكة) بـ **13/16/20/24/32/40/48/56/64**. السبب: حلّ تعارض السُّلّم مع نظام المسافات 8px — الانسجام الرأسي الكامل أهم من النقاء الرياضي. الشبكة تعلو على النسبة. (§2)
- **المسافة بين الأقسام كسُلّم لا رقم موحّد:** 128px الجامدة أُبدِلت بـ 112px افتراضي / 128px للأقسام الكبرى / 80px للفرعية. السبب: توحيد الفاصل على رقم واحد يقع في فخّ التناظر الآلي المولّد؛ التنويع مقصود. (§5.10)
- **نظام الظلال المتدرّج (تجاوز «صفر ظلال»):** «صفر ظلال» كانت مبالغة دفاعية؛ المشكلة الظل الافتراضي الكسول لا الظل نفسه. الحلّ المعتمد: نظام رباعي من طبقتين بلون الحبر (`--shadow-card/hover/float/bar`) مع ظل الشريط المعكوس لأعلى، blur ≤ 40px، بلا لون. (§1، §7.3)
- **أبعاد البطاقة (3:2 بدل 4:3):** 4:3 قصيرة تميل للمستطيل الجامد. المعتمد: 3:2 (368×245px)، صورة full-bleed، حشو غير موحّد (20/20/18)، شارة داخل منطقة النص، hairline فاصل، وصف مقصوص سطرين، تكبير صورة scale(1.03) عند hover. (§5.5)
- **معالجة الصور (saturate 0.92) — مرفوضة:** رُفض خفض التشبّع صراحةً؛ خضار الطزاجة الزاهي هو مصدر الحيوية والصدق. المعتمد: الصور بألوانها الطبيعية الكاملة، مع طبقة `::after` خفيفة فقط لتثبيت النص/الشارة فوق الصورة. (§5.8)
- **عمود القراءة والتثبيت اللا-متماثل:** أُضيف `--measure` = 640px (≈65 محرفاً) لكتل النص، ونظام تثبيت بعنصر واحد متناوب الجهة (زجزاج يقود التمرير)، مع ترجمة موبايل (إزاحة عمودية + نزيف واحد) وحماية `overflow-x: clip` بـ`margin-inline-end` سالب لا `transform`. (§11، §9)
- **لمسات الخبير:** hairlines بلون الحبر الشفاف (لا رمادي صريح)، الخطّ التحريري الكهرماني فوق العناوين، `:focus-visible` بدل `:focus`، أرضية `--img-placeholder` تحت الصور، `tabular-nums` للأرقام، تدرّج سماكة الحدود. (§7.5، §5.8)
